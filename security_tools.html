Wiz Security Platform

What is Wiz?

Wiz is a cloud security platform that provides comprehensive visibility and risk analysis across cloud environments. It helps organizations detect vulnerabilities, misconfigurations, compliance issues, and runtime threats in cloud workloads.

How Wiz works internally:

  • Agentless scanning: Wiz connects to cloud providers (AWS, Azure, GCP) via APIs to collect metadata about assets, configurations, and workloads without installing agents on hosts.
  • Cloud graph technology: Builds a real-time, interconnected model of your cloud infrastructure, mapping relationships between resources, identities, and network flows.
  • Risk prioritization: Uses context-aware algorithms to rank vulnerabilities based on exposure, exploitability, and business impact.
  • Continuous monitoring: Automatically rescans environments on configuration or workload changes to detect new risks.

Real-world usage:

  • Security teams get a unified dashboard for all cloud risks.
  • Automated compliance checks for standards like PCI-DSS, HIPAA.
  • DevOps can fix IaC misconfigurations before deployment.
Component Description
Agentless Scanning Uses cloud provider APIs (AWS, Azure, GCP) with read-only access to gather metadata about cloud resources (VMs, containers, IAM roles, etc.) without installing agents.
Cloud Graph Builds a real-time map of cloud assets and their relationships, including network topology, permissions, vulnerabilities, and runtime behavior to prioritize risk accurately.
Vulnerability and Misconfiguration Detection Scans for known CVEs in OS packages and third-party software, and detects cloud misconfigurations such as overly permissive S3 buckets or open security groups.
Runtime and Behavior Monitoring Integrates runtime telemetry to detect anomalies, suspicious activities, and zero-day exploits in workloads beyond static scanning.
Compliance and Governance Includes pre-built compliance policies (PCI-DSS, HIPAA, SOC2) and continuously audits environments to generate audit-ready reports.
Risk Prioritization & Remediation Guidance Assigns risk scores based on exploitability, exposure, and asset criticality; provides actionable remediation steps like patch instructions or configuration changes.
Common Commands & Interactions
  • Web UI: Dashboards, detailed views, scanning schedules, alert setup.
  • REST API: Automate data retrieval and integration with CI/CD or SIEM.
    curl -X GET "https://api.wiz.io/v1/vulnerabilities" -H "Authorization: Bearer <API_TOKEN>"
  • CLI / API clients: Use curl/Postman for scripting and automation.
  • Integration: Connect with Jira, ServiceNow, CI/CD pipelines for alerts and enforcement.

Snyk - Developer Security Platform

What is Snyk?

Snyk is a developer-first security platform focusing on identifying vulnerabilities in code, open-source dependencies, container images, and Infrastructure as Code (IaC) templates.

How Snyk works internally:

  • Dependency scanning: Integrates with package managers (npm, Maven, PyPI) to scan open-source libraries for known vulnerabilities.
  • Code scanning: Static Application Security Testing (SAST) analyzes your source code for security flaws.
  • Container scanning: Inspects Docker images for vulnerable layers.
  • IaC scanning: Validates Terraform, CloudFormation, Kubernetes YAML for security and compliance issues before deployment.
  • CI/CD integration: Snyk can be integrated into build pipelines to prevent vulnerable code from progressing.

Real-world usage:

  • Developers get actionable fix advice and automated pull requests.
  • Security teams track vulnerability trends across projects.
  • Prevent supply chain attacks by scanning dependencies early.

Infrastructure as Code (IaC) with Terraform

What is Terraform?

Terraform is an open-source IaC tool that lets you define and provision cloud infrastructure through declarative configuration files. It supports multiple providers like AWS, Azure, Google Cloud, Kubernetes, and many others.

How Terraform works internally:

  • Configuration files: You write infrastructure specs in HashiCorp Configuration Language (HCL), a human-readable syntax.
  • Execution plan: Terraform compares the desired state (configuration files) with the current real-world state and generates a plan of actions to achieve the desired state.
  • Apply changes: Applies the plan by creating, updating, or deleting resources via provider APIs.
  • State management: Maintains a state file that keeps track of managed resources, allowing incremental updates and drift detection.

Terraform example snippet (AWS EC2 instance):

provider "aws" {
  region = "us-east-1"
}

resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}

Why Terraform and IaC matter:

  • Automates and standardizes infrastructure provisioning.
  • Makes infrastructure reproducible and version-controlled.
  • Supports collaborative workflows in DevOps teams.
  • Enables infrastructure testing and validation (e.g., with Snyk IaC scanning).
Aspect Infrastructure as Code (IaC) with Terraform
Core Functionality Terraform enables automated provisioning, management, and versioning of infrastructure using declarative configuration files.
Architecture & Internals
  • Uses HashiCorp Configuration Language (HCL) for writing config files.
  • Terraform CLI parses configs and builds an execution plan.
  • State file maintains the current infrastructure snapshot.
  • Providers abstract API interactions with cloud platforms (AWS, Azure, GCP, etc.).
  • Supports modules for reusable infrastructure components.
Workflow
  1. Write infrastructure config in `.tf` files.
  2. Run `terraform init` to initialize providers and modules.
  3. Run `terraform plan` to preview changes.
  4. Run `terraform apply` to create/update resources.
  5. Use `terraform destroy` to remove resources.
Common Commands 
terraform init       # Initialize Terraform configuration
terraform plan       # Show execution plan
terraform apply      # Apply changes to infrastructure
terraform destroy    # Destroy all managed infrastructure
terraform fmt        # Format config files
terraform validate   # Validate configuration syntax
State Management
  • State file (`terraform.tfstate`) tracks resource metadata.
  • Supports remote state storage for team collaboration (S3, Azure Blob, etc.).
  • State locking prevents concurrent changes.
  • State drift detection to identify out-of-band changes.
